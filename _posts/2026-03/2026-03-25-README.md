## 什么是 Digest？

Digest 是针对单条专利的**深度分析流程**，与每日自动监控的 `daily/` 目录不同：
- **输入**: 用户指定的 Google Patents URL + 可选备注
- **处理**: 浏览器抓取 → LLM 分析翻译 → 图片处理 → Markdown 报告
- **输出**: 结构化的专利分析报告

## 目录结构

```
digest_report/
├── YYYYMM/                           # 年月文件夹
│   ├── {专利号}_{YYYYMMDD}_full.json     # 完整抓取数据（标题、摘要、说明书、权利要求、附图URL）
│   ├── {专利号}_{YYYYMMDD}_analysis.json # LLM 分析结果（中文翻译 + 技术分析）
│   ├── {专利号}_{YYYYMMDD}.jpg           # 附图合成图（JPG，80%质量）
│   ├── {专利号}_{YYYYMMDD}.md            # Markdown 分析报告
│   └── images/                           # 附图原图（可选）
│       └── USXXXXXXX_*.png
```

## 文件说明

| 文件类型 | 命名格式 | 说明 |
|---------|----------|------|
| `*_full.json` | `{patent_id}_{YYYYMMDD}_full.json` | 浏览器抓取的完整专利数据（原始文本 + 附图URL） |
| `*_analysis.json` | `{patent_id}_{YYYYMMDD}_analysis.json` | LLM 分析结果：title_cn, abstract_cn, analysis 四维度分析 |
| `*.jpg` | `{patent_id}_{YYYYMMDD}.jpg` | 专利附图合成图（4×3网格，JPG格式） |
| `*.md` | `{patent_id}_{YYYYMMDD}.md` | Markdown 格式分析报告 |

## 数据抓取规格

### 文本抓取限制

| 字段 | 限制 | 说明 |
|-----|------|------|
| abstract | 无限制 | 全部抓取 |
| description | ≤10,000 字符 | 移除 BRIEF DESCRIPTION，在 FIG. 处截断 |
| claims | ≤3,000 字符 | 超长截断 |

### 图片处理规格

- **单图画布**: 450×450 像素，白色背景
- **边框**: 440×440 区域，2px 浅灰色边框线
- **图片内容**: 430×430 像素，居中显示，保持比例缩放
- **网格布局**: 最多 12 张图，4 列 × 动态行数
- **输出格式**: JPG，80% 质量压缩

## LLM 分析输出格式

```json
{
  "title_cn": "中文标题（直译）",
  "abstract_cn": "中文摘要（直译）",
  "analysis": {
    "technical_field": "技术领域...",
    "background": "发明背景...",
    "overview": "发明总览...",
    "key_innovations": "核心创新点..."
  }
}
```

## 与 Daily 流程的区别

| | Daily | Digest |
|--|-------|--------|
| **触发方式** | 定时任务（周一至周五） | 用户手动指定 URL |
| **数据来源** | WIPO RSS | Google Patents |
| **分析深度** | 批量过滤 + 基础翻译 | 单条深度分析 |
| **图片处理** | 截图/缩略图 | 高清下载 + 网格合成 |
| **输出** | 每日汇总报告 | 单专利详细分析 |

## 相关脚本

- `digest/scripts/browser_fetcher.py` - 浏览器抓取专利数据和高清附图
- `digest/scripts/image_processor.py` - 附图下载和网格拼接
- `digest/scripts/llm_client.py` - MiniMax API 调用（Anthropic 兼容格式）
- `digest/templates/prompt_digest_v1.txt` - LLM Prompt 模板

## 注意事项

- 本目录仅存放生成的分析报告和输出文件
- 原始数据通过浏览器实时抓取，不依赖 RSS
- 图片采用 Google Patents 高清原图，非缩略图
- LLM 分析结合权利人公司产品进行应用场景推测
